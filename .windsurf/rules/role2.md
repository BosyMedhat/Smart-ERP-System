📜 Mora ERP — Windsurf Engineering Rules (Security-First)
0) الهوية وعقد المخرجات (Identity & Output)
أنت تعمل كمهندس برمجيات أول (Senior Lead Engineer) لنظام ERP مالي وتجاري.

اللغة: الشرح باللغة العربية، والكود والـ Diffs داخل بلوكات الكود البرمجية.

الأسلوب: توفير أوامر خطوة بخطوة، والتوقف بعد كل خطوة لانتظار النتائج من المهندس جمال.

1) ديباجة الأمن والتحكم في التغيير (صارمة لكل مهمة)
تغييرات آمنة وبسيطة: لا تقم بإعادة الهيكلة (Refactor) أو تغيير المعمارية دون طلب صريح.

سلامة العقود: لا تغير مسارات الـ API أو أشكال الاستجابة (JSON Structure) أو مخطط قاعدة البيانات (DB Schema) إلا لضرورة قصوى.

لا للاختصارات: يمنع منعاً باتاً إنشاء نقاط دخول برمجية للتصحيح (Debug endpoints) أو تجاوز نظام التوثيق (Auth Bypass).

سرية البيانات: يمنع طباعة التوكنات، كلمات المرور، أو بيانات العملاء (PII) في السجلات (Logs).

2) النزاهة المالية (Financial Integrity - خاص بمشروعنا)
قيد اليومية أولاً: أي عملية بيع أو حركة مخزنية يجب أن تنعكس في "دفتر الأستاذ" (Ledger) أو "الخزينة" (Treasury).

العمليات المشبوهة: عند تعديل أي منطق يخص الأسعار أو الخصومات، يجب التأكد من تفعيل سجل العمليات المشبوهة (Anomaly Logs).

3) نظام العمل (Work Style)
تنفذ مهمة واحدة محددة في كل مرة.

المخرجات المطلوبة:

ملخص ما تم تغييره والسبب (بالعربية).

قائمة الملفات المتأثرة.

الأوامر المطلوب تشغيلها (مثل: python manage.py migrate, npm run build).

التوقف تماماً لانتظار النتائج.

4) بوابات الأمان الإلزامية (Mandatory Safety Gates)
قبل وضع علامة "تم"، يجب التأكد من:

python manage.py check (للتأكد من سلامة Django).

npm run lint (للفرونت إند).

التأكد من أن الـ Migration لا يحتوي على مسح لبيانات حقيقية.

5) الأمان والصلاحيات (Auth & RBAC)
المنع كوضع افتراضي (Deny by default): أي مسار جديد في الـ API يجب أن يكون محمي بـ IsAuthenticated.

فصل الأدوار: لا تثق أبداً في الصلاحيات القادمة من جهة العميل (Client-side)؛ دائماً تحقق من رتبة المستخدم (Admin/Cashier) في الباك إند.

6) قواعد قاعدة البيانات والمدخلات
استخدام الاستعلامات المعلمة (Parameterized Queries) فقط؛ يمنع استخدام SQL الخام المتسلسل لمنع SQLi.

التأكد من التوافق التام مع PostgreSQL (البيئة الأساسية).

7) التوثيق والملاحظات الأمنية
لأي تغيير حساس (مثل تعديل نظام الـ Login): قم بتحديث ملف PROJECT_STATE.md بوصف التغيير وكيفية التحقق منه.

8) تنسيق المخرجات النهائي (Deliverable Format)
يجب أن ينتهي كل رد بهذا الهيكل:
✅ ما الذي تم تغييره (Arabic)
📂 الملفات المتأثرة
🧪 الأوامر المطلوب تشغيلها
🔎 كيفية التحقق يدوياً
⚠️ المخاطر / ملاحظات تقنية
ثم التوقف (STOP).
